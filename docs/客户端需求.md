校园助手 App (iOS 客户端) 需求文档

1. 需求背景
   1.1 目的
   本文档基于小组确定的“三页式”结构，明确“校园助手 App”的功能、页面布局和交互逻辑，作为 iOS 客户端开发的核心指导文件。
   1.2 核心结构
   应用采用底部 Tab Bar 导航结构，包含三个一级页面：
   日历/课表查询
   BBS 学校论坛
   个人页面/登录

2. 竞品调研

竞品名称 核心功能 突出优势 主要缺陷

---

滑铁卢大学 App 学校事件日历，课表查询，成绩查询，活动预约 页面简洁，信息直观 功能局限，日历和课表查询都缺少图示，只有文字
南京邮电大学 App 课表、校车以及一卡通查询，场地预约，图书馆检索书目，空教室查询(beta)，学校事件要闻 页面简洁，排版合理，核心功能齐全 学校事件要闻界面排版简单，只有文字，没有背景，缺少校园学生交流论坛等功能
香港大学 App 学生卡二维码（可以代替刷学生卡），香港大学地铁站实时排队情况，课程签到，学校新闻，校园地图 学校官方 App，功能齐全，比如学生二维码可以代替学生卡功能非常方便 过于官方，没有论坛等一些轻松的功能；功能虽然很多但是大部分用不上，核心功能只有学生二维码和地铁站实时排队情况
同济大学 App 二手市场，课程表，成绩查询，校园组队，选课情报 成绩（绩点、体测等）直接查询（避免学校多层查询）；界面简洁可爱 没有交流论坛，依赖外部论坛

3. 目标用户
   3.1 用户画像
   主要用户：在校大学生（本科生、研究生）
   次要用户：学校管理人员、教师
   3.2 用户需求分析

用户类型 核心需求 使用场景

---

学生 快速查看课表、参与校园讨论、管理个人信息 课前查看教室、课后交流学习、查询成绩等
管理员 管理用户信息、维护论坛秩序、发布重要通知 日常管理、应急通知、数据分析
点击图片可查看完整电子表格
3.3 用户角色与权限
3.3.1 角色定义
3.3.1.1 学生用户
基础权限：查看个人课表、参与论坛讨论、管理个人资料
限制：无法管理其他用户、无法删除他人内容
3.3.1.2 管理员用户
基础权限：包含学生所有权限
管理权限：
用户管理：创建/删除/禁用用户账号
内容管理：删除不当论坛内容、置顶重要帖子
通知管理：发布全校通知
数据查看：查看用户活跃度统计
特殊权限：后台管理界面访问
3.4 权限矩阵

功能模块 学生权限 管理员权限

---

课表查看 ✓ ✓
课表编辑 ✗ ✓（批量导入）
发布帖子 ✓ ✓
删除帖子 ✗ ✓（所有帖子）
用户管理 ✗ ✓
系统设置 ✗ ✓
点击图片可查看完整电子表格

4. 目标与愿景
   4.1 产品目标
   在训练营期间，实现一个具备优秀用户体验和流畅动画的 iOS 客户端，核心功能围绕学生的学习和信息获取需求展开，并包含基础的用户角色权限管理。
   4.2 交付物要求
   实现三个一级 Tab Bar 页面
   实现课表查询模块的单日/多日视图切换和二级详情页
   实现论坛模块的帖子列表展示和内容展开
   实现个人页面的登录状态管理和角色识别

6.1 课表/日历模块
6.1.1 核心功能
6.1.1.1 课表展示
视图模式：
周视图：默认显示当前周课表，左右滑动切换
日视图：点击某天查看当日详细课程安排
月视图：概览整月课程分布
显示内容：
课程名称（主标题，加粗显示）
上课时间（开始-结束时间）
教室位置
任课教师
课程类型（必修/选修/实验等，用不同颜色标识）
6.1.1.2 课程详情查看（单击功能）
触发方式：用户单击课表中的任意课程格子
详情内容：
课程基本信息：课程名称、课程代码、学分
详细时间：周次、节次、持续时间
地点信息：教学楼、教室号、教室类型（多媒体/普通）
教师信息：教师姓名、职称、联系方式（可选显示）
考核方式：考试/考查、成绩构成比例
课程评论：学生可以对该课程发表评论
交互操作：
添加到日历
提醒设置（课前提醒：15/30/60 分钟）
分享课程信息
返回课表
6.1.1.3 课表管理
选课：选课页面
个性化设置：
课程颜色自定义
隐藏/显示特定课程类型
6.1.2 业务规则
学生只能管理自己的课表
管理员可以批量导入课表数据
课表冲突检测：同一时间段不能安排两门课程
6.1.3 界面原型要求
周视图采用 7 列（周一到周日），每行代表一个时间段
课程格子采用卡片式设计，显示关键信息
点击课程后弹出模态框或新页面展示详情
颜色标识：
必修课 - 蓝色
选修课 - 绿色
实验课 - 橙色
公共课 - 紫色

6.2 论坛模块
6.2.1 核心功能
6.2.1.1 板块管理
预设板块：
课程讨论区：按学院/专业分类
生活服务：食堂、宿舍、校园设施
二手交易：教材、电子产品、生活用品
校园活动：社团活动、讲座、比赛
求助专区：学习帮助、生活求助
闲聊灌水：轻松话题讨论
动态板块：管理员可创建临时板块（如迎新专区、考试周专区）
6.2.1.2 内容发布与互动
发帖功能：
标题（20 字以内）、内容（富文本编辑器）
附件上传：图片（最多 9 张）、文件（PDF/Word 等，50MB 以内）
话题标签：#考试# #兼职# #失物招领# 等
互动功能：
点赞/踩：每帖限 1 次
评论：支持对帖子发表评论
评论互动：对评论的点赞、举报和删除（发出评论者）
评论框界面：显示评论者头像、ID 和评论时间
分享：分享到微信、QQ 等
收藏：保存感兴趣的内容
内容管理：
按时间、热度、最新回复排序
搜索功能：按标题、内容、用户、标签搜索
内容过滤：展示全校内容、仅本学院内容
举报功能：违规内容举报
6.2.2 业务规则
发帖频率限制：普通用户每小时最多 5 帖
内容审核：违禁词敏感词检测过滤，管理员人工审核
举报处理：收到 3 次举报自动隐藏，管理员 24 小时内处理
6.2.3 界面原型要求
论坛首页显示热门板块和最新帖子
帖子列表显示标题、作者、发布时间、回复数
详情页采用对话式设计，主帖在上，评论在下
支持夜间模式，保护用户视力

6.3 活动模块
6.3.1 核心功能
6.3.1.1 管理员发布活动
活动信息包括：活动标题，主要内容，时间地点，主办单位
管理员可以查看已发布活动的学生报名信息表并对所有报名学生进行参与统计
6.3.1.2 学生报名参加活动
提供报名与取消报名活动功能
学生报名或取消报名后，在该名学生的“我的活动”列表中显示活动信息和状态
6.3.2 业务规则
“我的活动”列表中展示的活动状态有：已报名、已取消报名、已参加、未参加
发布活动时需要进行敏感词检测，若检测到敏感词，禁止发布
活动举办时间需要在合理范围内（不得早于当前时间，结束时间不得早于开始时间）

6.4 个人/账号模块
6.4.1 核心功能
6.4.1.1 个人资料管理
基本信息：姓名、学号、学院、专业、年级、班级
头像上传：支持拍照和相册选择
个人简介：100 字以内
联系方式：
手机号（加密显示）
邮箱（用于找回密码）
微信/QQ（可选填写）
6.4.1.2 账号安全
登录方式：
学号+密码（初始密码由管理员分配）
手机验证码登录
第三方登录（微信、QQ，需绑定学号）
安全设置：修改密码
6.4.1.3 个性化设置
通知设置：
课前提醒开关
论坛新回复提醒
系统通知开关
隐私设置：
谁可以看到我的课表（仅自己/同班同学/所有人）
是否公开手机号
主题设置：浅色/深色模式
6.4.1.4 数据统计
学习统计：本周上课节数
本月论坛活跃度
收藏的课程/帖子数量
6.4.1.5 个人信息展示（用于其他用户查看）
个人基本信息展示
数据统计展示
6.4.2 业务规则
学号作为唯一标识，不可修改
敏感信息（手机号、邮箱）脱敏展示
个人资料修改需要身份验证
管理员可重置用户密码，但不能查看用户密码
6.4.3 界面原型要求
个人主页采用卡片式布局
设置项分组显示（账号、隐私、通知等）
6.4.3.1 活动列表展示
展示校园内所有活动，支持筛选与搜索
功能点：
活动卡片展示：封面图、标题、时间、地点、人数组合
活动状态：报名中 / 已满 / 已结束
搜索活动
活动类型筛选（讲座 / 社团 / 竞赛）
交互：
点击进入活动详情
下拉刷新
6.4.3.2 活动详情
展示活动的完整信息
展示信息：
标题 + 封面图
活动时间、地点
活动介绍（文本）
报名人数 / 限额
主办方信息
是否需要签到
功能点：
报名 / 取消报名
收藏活动
分享活动（通过 ID）
提示规则：
人数已满不可报名
报名未开始不可报名
活动结束后不可操作
6.4.3.3 活动报名
用户填写必要信息完成报名
表单字段（自动填充部分来自个人资料）：
姓名
学号
学院/专业
手机号（可选）
流程：
登录检查
人数检查
表单提交
显示“报名成功”
6.4.3.4 我的活动
展示与用户相关的活动
内容：
我报名的活动
我收藏的活动
我的签到记录
状态标识：
即将开始
已报名
已结束
